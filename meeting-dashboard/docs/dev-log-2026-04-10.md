# Meeting Dashboard 開発ログ (2026-04-10)

## 概要
Weekly Meeting Agenda ダッシュボード (Flask + BigQuery + Chart.js) の機能改善・バグ修正セッション。

---

## 完了した作業

### 1. Excelダウンロード改善
- **Summary シート削除**: RawData のみに変更
- **列名変更**: `Month` → `Month(Calendar)`, `ym` → `ETD(445)`
- **不要列削除**: `TEU_score`, `TEU_lpa`, `TEU_with_cm1`
- ダウンロードデータは「更新」ボタンのキャッシュと同一データを使用

### 2. PH タブ統合 (MNL + MIP)
- MNL/MIP を個別タブから PH タブに統合
- `AREA_AUTO_SUM = {"KR": ["JPC_KR","JPN_KR"], "PH": ["MNL","MIP"]}`
- `HIDDEN_TABS = new Set(["MNL","MIP"])` で子タブ非表示
- Prospect 入力は MNL/MIP それぞれ横並びで表示

### 3. 積み上げ棒グラフ (Stacked Bar Charts)
- Monthly/Weekly Trend で MNL + MIP を積み上げ表示
- `stack: 'actual'` / `stack: 'prospect'` でグループ化
- 各サブエリア (MNL, MIP) のバー内に個別数値を表示 (14px)
- 合計値はバー上部に表示 (line型ダミーデータセットで Y軸影響なし)
- 凡例 (エリア名) は 16px
- `stacked: hasSubStack` で非サブエリアチャートは通常表示を維持

### 4. 予測 (Prediction) バグ修正
- **MNL/MIP 同値問題**: `PREDICT_QUERY` に POD カラム追加、`_filter_area_no_pod` で POD フィルタリング対応
- **月間予測が高すぎ**: キャップ追加 (`同月過去最大 × 1.2`)
- **月間予測が実績以下**: フロア追加 (`max(pred, curr_actual)`)
- **週間予測が実績と同値**: 最低成長率追加 (`effective_rate = max(week_rate, 0.10)`)

### 5. POL Count AI プロンプト強化
- `_pol_new`, `_pol_lost`, `_pol_shipper_top3`, `_pol_shipper_prev_top3` をプロンプトに追加

## アーキテクチャメモ

### AREA_AUTO_SUM パターン
```python
AREA_AUTO_SUM = {"KR": ["JPC_KR", "JPN_KR"], "PH": ["MNL", "MIP"]}
```
親エリアは子エリアの合算で自動計算。API レスポンスに `sub_areas` と `sub_area_names` を含める。

### SUB_AREA_DEFS (POD ベースフィルタリング)
```python
SUB_AREA_DEFS = {"MIP": {"parent": "PH", "pod": ["MIP"]}, "MNL": {"parent": "PH", "pod": ["MNL"]}}
```

### Chart.js 積み上げ棒グラフ
- 各サブエリアを `stack: 'actual'` / `stack: 'prospect'` で分離
- 合計ラベルは `type: 'line', showLine: false, pointRadius: 0` のダミーデータセット
  (透明バーだと Y 軸が倍になるバグの回避策)

### 予測ロジック
```
月間: seasonal_index × baseline_TEU
  → cap: same_month_max × 1.2
  → floor: max(pred, curr_actual)
週間: week_remaining_rate (min 10%) で伸び保証
```

## トラブルシューティング履歴

| 問題 | 原因 | 解決策 |
|------|------|--------|
| ローカル変更が反映されない | 古い Python プロセスが残存 | `Get-Process python \| Stop-Process -Force` で kill |
| sub_areas が API レスポンスにない | 上記と同じ (旧コード実行中) | サーバー再起動 |
| Y 軸が大きすぎる | 透明ダミーバーが同スタックで値を倍加 | line 型ダミーに変更 |
| MNL/MIP 予測が同値 | PREDICT_QUERY に POD なし | POD を SELECT/GROUP BY に追加 |
| 月間予測 2,020 vs 実績 1,324 | remaining_rate が過大 | cap (past max × 1.2) 追加 |
| 週間予測 = 実績 | week_rate が低すぎ | min 10% growth rate 追加 |
